원인

BTL 패치된 di_server.jar 는 SP 호출 시 {? = call name(?)} (FUNCTION 호출 방식) 을 사용. PG 에서 같은 이름이 PROCEDURE 로 정의되어 있으면 호출 실패.

운영 서버에 dev 의 FUNCTION 정의를 적용했어야 하는데, 충돌 회피로 미적용한 routine 들이 이 에러를 일으킴.

진단

-- 같은 이름의 routine 종류 확인
SELECT p.proname,
       CASE p.prokind WHEN 'f' THEN 'FUNCTION' WHEN 'p' THEN 'PROCEDURE' END AS kind,
       pg_get_function_identity_arguments(p.oid) AS args
FROM pg_proc p
JOIN pg_namespace n ON n.oid = p.pronamespace
WHERE n.nspname = '[스키마]' AND p.proname = '[함수명]';

PROCEDURE 만 나오면 충돌 원인 확정. FUNCTION 으로 재정의 필요.
